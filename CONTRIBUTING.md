# Contributing to The Complete macOS VoiceOver Guide

Thank you for your interest in contributing! This guide aims to be a comprehensive, accurate, and accessible resource for VoiceOver users.

## Ways to Contribute

### Report Issues

- Typos or grammatical errors
- Incorrect or outdated commands
- Missing information
- Unclear explanations
- Broken formatting

Please open an issue with:
- The chapter/section affected
- What's wrong
- What it should say (if you know)

### Submit Corrections

For small fixes:
1. Fork the repository
2. Make your changes
3. Submit a pull request

### Add Content

For new content or major changes:
1. Open an issue first to discuss
2. Fork the repository
3. Create a new branch
4. Make your changes
5. Submit a pull request

## Writing Guidelines

### Style

- Write in clear, simple language
- Avoid jargon unless necessary (and define it when used)
- Use active voice when possible
- Keep sentences concise
- Use consistent formatting

### Formatting

- Use Markdown formatting
- Commands should be in **bold**: **VO + Space**
- Key combinations use + between keys: **Control + Option + Right Arrow**
- Use tables for command lists
- Use proper heading hierarchy (# for chapter, ## for sections)

### Accessibility

Remember that many readers use screen readers:
- Use descriptive headings
- Keep table structures simple
- Avoid relying solely on visual formatting for meaning
- Include alt text if adding images

### Technical Accuracy

- Test commands on the latest macOS version
- Note version-specific differences
- Link to Apple's documentation when appropriate
- When uncertain, say so

## Code of Conduct

- Be respectful and inclusive
- Welcome newcomers
- Accept constructive criticism gracefully
- Focus on what's best for the community

## Testing Changes

Before submitting:
1. Build the EPUB locally to check formatting
2. Review in an EPUB reader
3. If possible, test with VoiceOver

## Questions?

Open an issue with your question, and we'll do our best to help.

Thank you for helping make this guide better!
